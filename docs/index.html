Why CapyDeploy Exists

CapyDeploy was born from a real limitation encountered during game development. While working with the Steam Deck and Valve's official SteamOS Devkit Client, a significant constraint became apparent: the tool is exclusively designed for SteamOS devices.

As the handheld gaming market expanded with devices like the ROG Ally, Lenovo Legion Go, and Linux-based systems running Bazzite or ChimeraOS, the need for a universal, platform-agnostic deployment solution became clear.

CapyDeploy fills this gap â€” providing game developers and enthusiasts with a streamlined way to deploy builds to any handheld device, regardless of operating system, while maintaining the simplicity and reliability expected from professional development tools.

Architecture

Hub-Agent pattern with WebSocket as the primary communication channel.

Hub

Your PC

  • Discovers Agents via mDNS
  • Opens WebSocket connection
  • Sends binary chunks
  • Stores auth tokens
  • Sends simple orders (Agent does the rest)

Agent

Target Device

  • WebSocket server on port 9999
  • Receives binary chunks
  • Creates Steam shortcuts
  • Applies artwork
  • Handles operations autonomously

Connection Flow

1

Agent broadcasts via mDNS

2

Hub connects via WebSocket

3

6-digit pairing code

4

Deploy games!

WebSocket API

All communication happens over WebSocket at ws://agent:9999/ws

Primary Protocol HTTP endpoints are deprecated

  "payload": { ... },
  "error": { "code": 400, "message": "..." }
}

RPC Messages

Request/response operations
get_info â†’ info_response Get Agent details and capabilities
get_steam_users â†’ steam_users_response List Steam users on device
list_shortcuts â†’ shortcuts_response List shortcuts for a user
create_shortcut â†’ operation_result Create a Steam shortcut
apply_artwork â†’ artwork_response Apply artwork to shortcut
delete_game â†’ operation_result Delete game completely (Agent handles everything)
delete_shortcut â†’ operation_result Delete a shortcut by appID
restart_steam â†’ steam_response Restart Steam client

File Upload

Binary chunk transfers

Binary Chunk Format

[4-byte header length][JSON header][binary chunk data]

JSON header:
{
  "id": "msg-id",
  "uploadId": "upload-abc123",
  "filePath": "data/game.pak",
  "offset": 0,
  "checksum": "sha256..."
}

Initialize upload session

Real-time progress events

Push Events

Real-time notifications from Agent
upload_progress Real-time upload progress (bytes, speed, ETA)
operation_event Operation progress (delete, install, restart)
shortcuts_changed Shortcut list was modified

  "type": "delete",      // "delete", "install", etc.
  "status": "progress",  // "start", "progress", "complete", "error"
  "gameName": "My Game",
  "progress": 50,
  "message": "Reiniciando Steam..."
}

Installation

Get CapyDeploy running in a few steps.

1

Install Prerequisites

# Install Go 1.24+ and Wails CLI
go install github.com/wailsapp/wails/v2/cmd/wails@latest
wails doctor
2

Clone and Build

git clone https://github.com/lobinuxsoft/capydeploy
cd capydeploy
git submodule update --init --recursive

# Build Hub (your PC)
cd apps/hub && ./build.sh

# Build Agent (handheld)
cd apps/agent && ./build.sh
3

Run and Pair

# On handheld
./capyagent
# Shows 6-digit pairing code

# On PC
./capyhub
# Enter the pairing code â†’ Done!

//go:build darwin

package steam

func getBaseDir() (string, error) {
    home, _ := os.UserHomeDir()
    steamPath := filepath.Join(home,
        "Library",
        "Application Support",
        "Steam")
    return steamPath, nil
}

2 Steam Controller 

//go:build darwin

func (c *Controller) Start() error {
    return exec.Command("open", "-a", "Steam").Run()
}

func (c *Controller) IsRunning() bool {
    out, _ := exec.Command("pgrep", "steam").Output()
    return len(out) > 0
}

func (c *Controller) Shutdown() error {
    return exec.Command("osascript", "-e",
        `quit app "Steam"`).Run()
}

Required Controller Methods

Start() Launch Steam
Shutdown() Graceful close
Restart() Full restart
IsRunning() Process check
IsGamingMode() Console mode
EnsureRunning() Start if needed
